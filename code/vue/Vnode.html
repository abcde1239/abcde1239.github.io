VNode

vnode 到真实 DOM 在 vue 中采用了 patch 方法，其中先判断了其是否为普通 vnode 或描述组件的 vnode，然后再进行递归直到到达普通 vnode 后，开始进行渲染。

Q:开发 vue 项目中，我们经常采用组件化设计，那么，组件是否一定是越细越好呢？

A:并非完全。当组件过于精细时，会导致 vnode 渲染时进行大量递归，影响性能。

Q:vue 的底层框架设计中，vnode 是什么，为什么会使用 vnode？

A: vnode，即 virtual node（虚拟节点），是 Vue 对真实 dom 的抽象表示，vnode 使得可以在内存中构建虚拟 dom 树，并由此描述 ui 结构，状态。其最核心的设计目的是：避免直接操作浏览器 DOM，提高性能，同时提供了跨平台能力。

关于为什么使用 vnode 的详细解释

1.性能优化

dom 操作代价较高（造成重排 重绘 以及回流，影响性能）。Vnode 在内存中更新通过使用 diff 算法->最小化 patch->减少真实 dom 操作

2.状态与视图分离

数据变化->更新 vnode->patch->DOM，不直接操作浏览器 api，更易于维护测试

3.支持跨平台

vnode 可以与平台渲染器解耦，使其在小程序，canvas，WebGL 上也能渲染。

4.抽象化

在 vue 中，vnode 将真实 DOM 抽象为内存中可操作的虚拟节点，使开发者更注重 UI 结构和数据变化

Q:解释 vnode 的挂载和更新，解释 diff 算法

vnode 使用 mount 和 patch，判断为挂载后，先判断 vnode 的 type，然后由处理 props->处理 children->插入 dom;vnode 判断为更新时，先比较 type，不同则卸载旧并挂载新，相同则复用原来的 dom，并进行更新 props：正常修改，删除，添加，最后更新 children。

children 的更新

Vue 通过 VNode 对 DOM 操作进行预处理以节约性能，分为初次挂载和更新两种情况。更新时，文本子节点直接覆盖，null 保持不变；而 VNode 或插槽子节点则走严格 diff：头尾扫描快速跳过相同节点，key 用于复用、删除或新增，LIS（最长递增序列） 优化确保最少移动。

diff 算法概述：头尾扫描+key 映射+LIS 优化移动
